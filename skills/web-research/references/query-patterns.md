# Query patterns for technical research

Use 4-6 genuinely different queries, not paraphrases of one query.

| Angle | Best for | Example |
| --- | --- | --- |
| Official docs | config, supported behavior, definitions | `official documentation for Model Context Protocol streamable HTTP stdio` |
| Source repo | real file layout, implementation details | `openai codex skills loader .agents/skills GitHub` |
| Release notes | recent changes, migration risk | `Model Context Protocol spec changelog streamable HTTP` |
| Comparison | build-vs-buy, tradeoffs | `Exa vs Perplexity for technical research` |
| Caveats | edge cases, disagreement, limits | `Claude Code MCP project scope approval .mcp.json` |

Good bundle order:
1. official docs
2. source repo
3. release notes
4. comparison
5. caveats

Avoid:
- near-duplicate wording
- broad questions with no product/version names
- synthesized answers without reading linked sources
